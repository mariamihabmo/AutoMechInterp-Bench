# Stage-1 Schema Reference

## `protocol.yaml`
Required top-level keys:
- `protocol_id`: string
- `protocol_version`: string
- `frozen`: bool (must be true)
- `unit_of_work`: mapping
- `execution_grid`: mapping
- `control_policy`: mapping
- `stage_gates`: mapping
- `statistical_policy`: mapping
- `claim_budget`: mapping
- `language_policy`: mapping

### `unit_of_work`
- `task_id`
- `model_id`
- `model_spec` (recommended for fallback-safe generation):
  - `n_layers`: int > 0
  - `n_heads`: int > 0
- `dataset_id`
- `metric_id`
- `clean_corrupt_definition`

### `execution_grid`
- `seeds`: list[int]
- `prompt_variants`: list[str]
- `resample_ids`: list[int]
- `methods`: list[str], minimum length 2

### `control_policy`
- `required_families`: list[str], must include:
  - `wrong_position`
  - `wrong_layer`
  - `random_component`
  - `mismatched_source`
- `max_control_abs_mean`: float

### `stage_gates`
- `min_causal_effect`: float
- `min_specificity_ratio`: float
- `min_robustness`:
  - `seed`: float
  - `prompt_variant`: float
  - `resample`: float
- `max_method_sensitivity_std`: float
- `require_confirmatory_ci_excludes_zero`: bool

### `statistical_policy`
- `alpha`: float
- `fdr_q`: float
- `min_effect_floor`: float
- `multiplicity_method`: string

### `claim_budget`
- `max_total_claims`: int > 0
- `max_claims_per_task`: int > 0

### `language_policy`
- `forbidden_without_pass`: list[str]

## `hypothesis.jsonl`
One JSON object per line. Required fields:
- `hypothesis_id`
- `protocol_id`
- `task_id`
- `model_id`
- `metric_id`
- `claim_text`
- `candidate_components` (list[dict])
- `predicted_effect_direction` in `{increase,decrease}`
- `predicted_min_effect`
- `predicted_specificity_ratio`
- `expected_failure_modes` (list[str])

Forbidden pre-verification fields:
- `accepted`
- `passed`
- `verdict`

### Agent-output ingestion (`generate-hypotheses`)
The CLI command can ingest:
- JSON list of hypothesis objects
- JSON object with one of: `hypotheses`, `items`, `candidates` (list of objects)
- JSONL with one hypothesis object per line

It auto-fills frozen fields from `protocol.yaml`:
- `protocol_id`
- `task_id`
- `model_id`
- `metric_id`

Supported aliases:
- `mechanistic_claim` -> `claim_text`
- `components` -> `candidate_components`
- `predicted_effect_size` -> `predicted_min_effect`
- `specificity_ratio` -> `predicted_specificity_ratio`
- `failure_modes` -> `expected_failure_modes`

## `agent_output.json` (generated by `generate-agent-output`)
Top-level shape:
- `agent_metadata`: object
- `hypotheses`: list of hypothesis proposal objects

Generated hypothesis proposal fields:
- `hypothesis_id`
- `mechanistic_claim`
- `components`
- `predicted_effect_direction`
- `predicted_effect_size`
- `specificity_ratio`
- `failure_modes`

`generate-agent-output` supports two source modes:
- `template`: hypotheses synthesized from protocol defaults + optional component catalog
- `exploration`: hypotheses derived from `--exploration-input` findings

Exploration input accepted shapes:
- JSON list of finding objects
- JSON object with `findings`, `hypotheses`, `results`, or `items` list

Each exploration finding should include at minimum:
- one of `components` / `candidate_components` / `top_components` (list of component objects)

Optional exploration fields used when present:
- claim text: `summary` / `claim_text` / `mechanistic_claim` / `hypothesis` / `description`
- effect: `metric_delta` / `effect_size` / `predicted_effect_size` / `treatment_effect_mean`
- specificity: `specificity_ratio` / `control_ratio`
- direction: `direction` / `predicted_effect_direction`
- failure modes: `failure_modes` / `expected_failure_modes`

## `evaluation_result.json`
Top-level required:
- `protocol_id`
- `protocol_sha256`
- `hypothesis_results` (list)

Each hypothesis result requires:
- `hypothesis_id`
- `raw_cells`

Each `raw_cell` requires:
- `seed`: int
- `prompt_variant`: str
- `resample_id`: int
- `method`: str
- `slice`: `exploratory|confirmatory`
- `treatment_effect`: float
- `control_effects`: mapping for all required control families
- `runner_id`: str
- `runner_version`: str
- `pipeline_sha`: str
- `model_ref`: str
- `dataset_seed`: int
- `prompt_template_id`: str

## `manifest.json`
Must contain SHA256 digests for:
- `protocol.yaml`
- `hypothesis.jsonl`
- `evaluation_result.json`

## Evaluator output (`result.json`)

Each claim report includes:
- `evidence_tier`
- `passed`
- `checks` for backward compatibility
- `gate_outcomes` with canonical values `pass`, `fail`, or `not_evaluated`
- `failed_checks`
- `not_evaluated_checks`
- `metrics`
